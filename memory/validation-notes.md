# 边界校验学习记录：上传视频表单

## 1. 为什么需要边界校验

上传视频功能不能只依赖用户“正常填写”。

用户可能会提交：

```txt
空标题
超长标题
超长介绍
null / undefined / NaN 这类无意义文本
过大的视频文件
过大的封面图片
伪装成图片或视频的其他文件
不支持的文件格式
```

所以需要做边界校验。

## 2. 前端校验和后端校验的区别

### 2.1 前端校验

前端校验主要是为了用户体验。

例如：

```txt
标题最多 120 个字
介绍最多 1000 个字
封面最大 5MB
视频最大 500MB
```

用户在页面上就能尽早看到提示。

但是前端校验不能作为真正安全保障，因为用户可以绕过浏览器页面，直接调用后端 API。

### 2.2 后端校验

后端校验才是真正可信的校验。

无论用户从哪里发请求，最终都必须经过后端。

所以本项目同时做了：

```txt
前端校验：提升体验
后端校验：保证安全和数据质量
```

## 3. 当前项目的校验规则

| 内容 | 规则 |
|---|---|
| 标题 | 必填，最多 120 个字 |
| 标题特殊值 | 不能是 null、undefined、NaN |
| 介绍 | 可选，最多 1000 个字 |
| 介绍特殊值 | 不能是 null、undefined、NaN |
| 视频 | 必填，最大 500MB |
| 视频格式 | mp4、webm、mov |
| 封面 | 必填，最大 5MB |
| 封面格式 | jpg、jpeg、png、webp |

## 4. 后端相关文件

```txt
backend/src/upload.js
backend/src/videos.routes.js
```

### 4.1 upload.js

`upload.js` 主要负责文件上传的基础限制：

```txt
multer 临时保存文件
限制文件总大小
判断视频 MIME 类型
判断图片 MIME 类型
判断视频扩展名
判断图片扩展名
```

### 4.2 videos.routes.js

`videos.routes.js` 主要负责业务字段校验：

```txt
标题是否为空
标题是否过长
介绍是否过长
是否填写 null / undefined / NaN
视频是否存在
封面是否存在
视频大小是否超过 500MB
封面大小是否超过 5MB
```

## 5. 前端相关文件

```txt
frontend/app/page.js
```

前端表单做了这些限制：

```txt
input maxLength
textarea maxLength
file accept
提交前检查 FormData
显示友好的错误提示
```

前端的 `accept` 可以让文件选择框优先展示指定格式：

```txt
视频：mp4、webm、mov
封面：jpg、jpeg、png、webp
```

但注意：`accept` 不是安全措施，后端仍然必须再次校验。

## 6. 关于 null 字符串

真正的 JavaScript `null` 和用户输入的字符串 `"null"` 不一样。

表单里用户输入的是字符串。

所以如果用户把标题写成：

```txt
null
```

它本质上是一个普通字符串。

本项目为了避免无意义数据，主动拦截这些字符串：

```txt
null
undefined
NaN
```

并且大小写不敏感，例如：

```txt
NULL
Undefined
nan
```

也会被拦截。

## 8. 重要原则

记住一句话：

```txt
前端校验是为了体验，后端校验才是真正安全。
```

数据库也可以做最后一层兜底，例如标题字段使用：

```sql
title VARCHAR(120)
```

但最好不要等数据库报错后再提示用户，而是在后端提前给出友好的错误信息。
